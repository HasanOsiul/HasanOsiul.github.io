01 / Project Overview

What is
Branch & Bound?

About This Project

This project is based on a Management Science reading about Branch and Bound (B&B) — one of the most important optimization algorithms used in operations research and decision science.

The algorithm solves complex combinatorial problems by systematically exploring the solution space while eliminating solutions that provably cannot be optimal — a technique called pruning.

The reading demonstrates the process using a pizza topping optimization problem: given a limited budget of 7 euros, how do we select toppings to maximize satisfaction? This deceptively simple problem becomes a gateway to understanding how real-world optimization works.

"Optimization is not about checking every possibility. It is about intelligently eliminating bad possibilities."

Core Concepts

Integer Programming

Variables restricted to 0 or 1: excluded or included. No fractions allowed.

Branching

Breaking one large problem into smaller subproblems at each variable.

Bounding

Calculating the best possible outcome a branch could ever achieve.

Pruning

Discarding branches whose bound cannot beat the current best solution.

Real-World Applications

  • Airline crew scheduling
  • Supply chain & logistics routing
  • Production planning & scheduling
  • Facility location problems
  • AI decision-making systems
  • Network design & optimization

sᵢ = satisfaction of topping i
cᵢ = cost of topping i

Evidence 04

B&B Workflow

① Relax Integer Constraints
② Solve LP Relaxation
③ Calculate Upper Bound
④ Branch on Fractional Variable
⑤ Evaluate Child Nodes
⑥ Prune if UB ≤ Best Known
✓ Optimal Solution Found

Evidence 06

Upper Bound vs Lower Bound

⬆ Upper Bound
The best possible outcome a branch could ever achieve, found by relaxing integer constraints (LP relaxation). If this is not better than what we already know, the branch is useless.
UB = LP Relaxation optimal
⬇ Lower Bound
The best feasible integer solution found so far. Acts as a benchmark — any branch whose upper bound does not exceed this gets pruned immediately.
LB = Best integer solution known
Pruning Rule: If UB(node) ≤ LB(best), prune that node.
Evidence 07

Why Pruning Works

① Evaluate a branch's upper bound
② UB = 8 < Current Best = 12
③ This branch cannot possibly beat what we know
④ PRUNE — eliminate entire subtree
⑤ Save computation on all descendant nodes
⑥ Algorithm converges faster to global optimum
In practice, B&B can explore <5% of the brute-force solution space on typical problems.

What I Learned

I gained a deep understanding of how integer programming works and why it is fundamentally harder than continuous optimization. The key insight is that restricting variables to binary values dramatically increases complexity — but B&B manages this elegantly through systematic search.

💡
Reflection

What Surprised Me

I was genuinely surprised that B&B does not need to evaluate every possibility. By computing upper bounds early, vast subtrees can be eliminated before being explored. Complex problems that would take hours by brute force can be solved in seconds.

Real-world impact spans everywhere from delivery routing to AI planning — the same algorithmic skeleton underlies modern logistics software used by companies like FedEx, Amazon, and airline networks.
